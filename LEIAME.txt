-- 0. Observações --
! Este arquivo contém informações básicas sobre o projeto. Peçamos que 
seja lido, sem falta, o arquivo "Relatório TP1.pdf". Isso porque nele
há informações detalhadas e de fácil leitura sobre o projeto. Acreditamos
que sim o arquivo de texto puro deve ser amplamente usado em programas 
de cunho profissional por conta de sua acessibilidade. Porém, com intuito 
de facilitar a interação alunos-monitor peçamos que seja feita a leitura
do pdf.

-- 1. Como usar --
! Versão C:
 -No terminal de sua máquina, use a seguinte sintaxe:
	gpalestra
	
! Versão Java:
 -Compile e rode o programa no Eclipse
	
! O programa irá ler do arquivo Palestrantes.txt e Palestras.txt a fim de
gerar um arquivo saída ao usuário contendo uma lista, em ordem cronológica,
das palestras a serem realizadas.
	

-- 2. Como Compilar --
! Recomenda-se usar o gcc nesta versão:
cristian@Jasmine-VM:~$ gcc --version
gcc (Ubuntu/Linaro 4.8.1-10ubuntu9) 4.8.1

! Para a versão C, vá até a pasta _versaoC e use makefile para compilar
automaticamente.

! Para a versão Java, não se esqueça de:

a) Possuir o JDK 8. Para instalar utilize os seguintes comandos:

sudo add-apt-repository ppa:webupd8team/java
sudo apt-get update
sudo apt-get install oracle-java8-installer

b) Possuir a IDE Eclipse. Faça o download em:

https://eclipse.org/downloads/

c) Incluir a pasta raiz do programa em workspace e criar um novo Java Project
com o mesmo nome da pasta extraida em seu computador a partir do zip.


-- 3. Versões --
0.1 - 	Primeira versão do gPalesta! Implementa os módulos básicos de leitura,
tratamento e persistência de dados referentes a palestrantes e palestra. Versão
em C cancelada (vide relatório em pdf), porém com implementação de módulo de testes
CUnit para funcionalidade de leitura e tratamento de dados. E versão Java com todos
os módulos implementados (programa funcional), todavia sem módulo JUnit.


-- 4. Informações Extras (informal) --
Referente a versão C:
Muita coisa pra fazer, só a parte de leitura já foi uma dor de cabeça. O programa
já ficou imenso só nesta parte. Porém já tem muita base pra se desenvolver
futuramente, como módulos para criação e manipulação de streams, listas e
dicionários. Como já foi feito a parte de entrada, a de saída seria análoga.
Há muita coisa repetida no código (parte de uso destes módulos base), mas
poderiam ser enxugadas futuramente. Algo foi feito na corrida (aprox. 21 horas
de trabalho total) porque há muita outas coisas pra se preocupar na UnB.


Considera-se que o 'TP1' de Programação Sistemática foi um sucesso.
